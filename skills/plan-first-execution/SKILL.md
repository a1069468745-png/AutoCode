# 先计划后实施

## 目标

确保在任务边界、依赖、进入门禁、验证路径和回写目标不清晰前，不启动任何实施工作。

## 强制检查项

实施前必须确认：

1. 当前任务已映射到明确的计划或任务片段
2. 当前任务边界明确
3. 上游门禁已满足
4. 当前任务进入门禁已检查并通过
5. 当前阶段自测边界已存在；若不存在，则先补测试边界或用例
6. 验证方式已知
7. 进度回写目标已知

## 任一检查失败时

- 停止实施
- 补齐缺失的计划、边界、门禁、测试边界或回写上下文
- 完成后再继续
