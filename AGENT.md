# AutoCode Agent 全局约束

## 1. 目的

本仓库采用严格的 `先计划、后开发、门禁驱动、可审计追溯` 工作模式。
任何在本仓库中工作的 Agent，都必须以仓库文档为唯一事实来源，不得凭经验、记忆或聊天上下文直接实施改动。

## 2. 最高优先级规则

1. 任何行动前，必须先加载最小必要上下文。
2. 任何开发前，必须先有计划。
3. 不得跳过当前阶段适用的门禁；研发阶段以任务边界、开发门禁和研发内自测为准，评审、审计、发布门禁在对应阶段生效。
4. 不得将上一次会话中的记忆视为当前事实来源。
5. 完成有意义的工作后，必须回写必要进度与状态；仅在形成持久性规则、架构或流程决策时回写决策记录。
6. 后续新增文档默认使用中文编写。
7. 治理、需求、设计、计划、质量、审计、进度类文档必须默认使用中文。
8. 仅在技术名词、接口名、协议名、字段名、代码标识不可避免时保留英文。
9. 任务开始前，必须先判断是否存在对应任务边界与进入门禁，只有满足要求后才允许真正开始实施。
10. 进入新阶段前，必须先判断当前阶段是否具备自测边界；如果没有，必须先补充测试用例、验证规则或验收边界，再进入阶段实施。
11. 所有实施必须在既定边界内执行，不得随意发挥或无依据扩张范围。

## 3. 权威来源顺序

当文档内容发生冲突时，按以下顺序判定：

1. `docs/context/current/`
2. `docs/plans/execution/`
3. `docs/specs/architecture/03-详细设计-LLD.md`
4. `docs/specs/architecture/02-概要设计-HLD.md`
5. `docs/specs/product/01-需求文档-PRD.md`
6. `docs/specs/analysis/`
7. `docs/governance/`

如果冲突无法安全消解，必须先停止实施并完成对齐。

## 4. 新会话强制启动流程

每个新会话必须按以下顺序读取：

1. `AGENT.md`
2. `CLAUDE.md`
3. `docs/indexes/00-总索引.md`
4. `docs/indexes/02-当前阶段索引.md`
5. `docs/indexes/03-当前任务索引.md`

只有完成以上步骤后，才允许按任务类型继续加载专项文档。

## 5. 强制工作顺序

仓库标准执行顺序如下：

1. 识别任务类型
2. 加载最小必要文档
3. 确认任务边界
4. 产出或更新计划
5. 检查门禁
6. 执行任务
7. 完成验证
8. 回写必要进度与状态记录
9. 再进入下一个任务

在第 1 至第 5 步未满足前，不得开始任何编码或实现类工作。

## 6. 硬边界

Agent 不得：

- 在没有计划或任务记录的前提下直接开发
- 在没有确认任务边界和进入门禁的前提下启动任务
- 在未更新架构或决策文档的前提下修改架构
- 对中高风险或非平凡改动跳过测试
- 在缺少阶段自测边界时直接推进阶段开发
- 在没有验证证据的情况下宣告完成
- 只把进度留在聊天记录中而不回写文档
- 无依据地更新无关文档或扩大文档修改范围
- 在可以小范围按需加载时一次性全量读取全部文档

## 7. 强制回写要求

完成有意义的工作后，必须按任务边界更新所有受影响且应当更新的文档，但不得无依据扩大更新范围。默认优先回写当前状态与必要进度；只有在形成持久决策时才要求补充决策文档。至少覆盖以下一类文档：

- `docs/context/current/`
- `docs/context/progress/`，如本次工作属于需要留痕的阶段性推进
- `docs/context/decisions/`，如本次工作形成持久规则、架构或流程决策
- `docs/plans/execution/`，如果范围或顺序发生变化
- `docs/specs/architecture/`，如果架构发生变化

## 8. 质量与审计

所有重要工作必须可以追溯到：

- 一个计划
- 一个任务边界
- 一个验证结果
- 一条进度更新记录

研发阶段必须保留最小可追溯链路，但不要求预先完成评审、审计或发布阶段材料；这些规则在对应阶段进入时按 `docs/quality/` 与 `docs/governance/` 生效。

## 9. Skill 优先规则

如果当前 Agent 环境支持工作区 Skill 发现机制，优先使用：

- `skills/session-bootstrap/`
- `skills/plan-first-execution/`
- `skills/progress-sync/`

如果当前环境无法直接使用 Skill，也必须遵循它们在 `CLAUDE.md` 和 `docs/governance/` 中对应的等价规则。
