# 文档总索引

## 1. 最小启动集

新会话默认先读以下文件：

1. `/AGENT.md`
2. `/CLAUDE.md`
3. `/docs/indexes/00-总索引.md`
4. `/docs/indexes/02-当前阶段索引.md`
5. `/docs/indexes/03-当前任务索引.md`

## 2. 文档分层

- `docs/indexes/`
  - 轻量导航、当前阶段、当前任务、最小加载入口
- `docs/governance/`
  - 开发治理、门禁、会话加载、审计规则
- `docs/specs/`
  - 产品、架构、分析类权威说明
- `docs/plans/`
  - 开发任务拆分、执行计划、后续阶段计划
- `docs/quality/`
  - 测试、评审、审计与发布质量规则
- `docs/context/`
  - 当前阶段、当前任务、最近进度、关键决策

## 3. 按任务类型导航

- 产品和范围：`docs/specs/product/`
- 架构与技术实现：`docs/specs/architecture/`
- 风险与问题分析：`docs/specs/analysis/`
- 执行和排期：`docs/plans/execution/`
- 质量与审计：`docs/quality/`
- 当前状态：`docs/context/current/`

## 4. 当前权威文档

- PRD：`docs/specs/product/01-需求文档-PRD.md`
- HLD：`docs/specs/architecture/02-概要设计-HLD.md`
- LLD：`docs/specs/architecture/03-详细设计-LLD.md`
- 执行计划：`docs/plans/execution/06-开发任务拆分与执行计划.md`

## 5. 会话建议

如果是新会话且任务未明确，先只读取：

- `AGENT.md`
- `CLAUDE.md`
- `docs/indexes/`
- `docs/context/current/`

只有在任务被明确后，再读取对应专项文档。
