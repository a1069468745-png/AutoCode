# 统一配置与环境变量规范

## 1. 目标

本规范用于固定一期 MVP 基础设施阶段的统一配置加载方式，确保后端服务、前端控制台和部署脚本使用一致的环境变量命名、覆盖优先级和敏感配置处理方式。

## 2. 后端配置分层

后端服务统一使用 `src/main/resources/application.yml` 作为基础模板。

约定如下：

- 默认通过 `SPRING_PROFILES_ACTIVE` 控制运行 Profile，未显式传入时默认为 `local`
- 基础模板只保留可安全公开的默认值
- 数据库、Redis、端口、日志级别、工作目录和模型配置全部允许由环境变量覆盖
- 后续若引入 `application-local.yml`、`application-dev.yml`、`application-prod.yml`，仍以基础模板为公共层，不直接复制整份配置

推荐覆盖优先级：

1. 启动命令显式传参
2. 操作系统环境变量
3. `.env` 或部署脚本注入变量
4. `application.yml` 默认值

## 3. 环境变量命名规则

### 3.1 通用规则

- 统一使用全大写字母与下划线
- 共享基础设施优先使用通用键：
  - `DB_HOST`
  - `DB_PORT`
  - `DB_NAME`
  - `DB_USERNAME`
  - `DB_PASSWORD`
  - `REDIS_HOST`
  - `REDIS_PORT`
  - `REDIS_PASSWORD`
  - `ROOT_LOG_LEVEL`
  - `APP_LOG_LEVEL`

### 3.2 服务端口规则

- 每个服务独立使用 `<SERVICE_NAME>_PORT`
- 当前端口规划：
  - `API_GATEWAY_PORT=18080`
  - `PROJECT_SERVICE_PORT=18081`
  - `CONTEXT_SERVICE_PORT=18082`
  - `CODEGRAPH_RUNNER_PORT=18083`
  - `HISTORY_INDEXER_PORT=18084`
  - `KNOWLEDGE_INDEXER_PORT=18085`
  - `LLM_GATEWAY_PORT=18086`
  - `DEV_AGENT_SERVICE_PORT=18087`
  - `WEB_CONSOLE_PORT=18088`

### 3.3 专项配置规则

- 本地工作区统一使用 `WORKSPACE_ROOT`
- 文档根目录统一使用 `DOCS_ROOT`
- 模型网关统一使用：
  - `LLM_PROVIDER`
  - `LLM_BASE_URL`
  - `LLM_API_KEY`
  - `LLM_MODEL_NAME`
  - `LLM_FALLBACK_MODEL`
  - `LLM_TIMEOUT_SECONDS`
  - `LLM_ENABLE_LOCAL_ONLY`

## 4. 敏感配置处理要求

- `DB_PASSWORD`、`REDIS_PASSWORD`、`LLM_API_KEY` 等敏感值不得写死在 Java 源码或前端源码中
- 示例文件可保留 `replace_me` 这类占位值，但不得提交真实密钥
- 正式环境应通过环境变量、CI/CD 密钥注入或部署平台密文能力传入
- 前端不得持有服务端密钥类配置

## 5. 前端环境变量规范

前端控制台统一使用 `frontend/web-console/.env.example` 作为示例模板。

约定如下：

- 仅允许暴露 `VITE_` 前缀变量
- 当前保留变量：
  - `VITE_APP_TITLE`
  - `VITE_APP_ENV`
  - `VITE_API_BASE_URL`
  - `VITE_REQUEST_TIMEOUT`
- 若后续新增前端配置，必须继续使用 `VITE_` 前缀
- 前端只能持有可公开配置，不得放入 API Key、数据库密码或内部管理口令

## 6. 与部署示例的关系

- `deploy/.env.example` 作为部署侧统一入口，负责基础设施组件、后端服务和 `web-console` 端口变量示例
- `frontend/web-console/.env.example` 作为前端侧统一入口
- 后续进入 `INF-05` 时，`docker-compose.yml` 中的服务占位配置应复用本规范中的命名，不再另起一套变量名

## 7. 阶段约束

- 本阶段只固定规范和模板，不接入企业配置中心
- 本阶段不要求按环境拆分多套实际配置文件
- 后续服务实现必须优先复用本规范，若需要新增变量，应先同步更新本文件与示例模板
