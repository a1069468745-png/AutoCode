# 数据库核心表结构与数据字典

## 1. 文档目标

本文档用于固化 `DB-01` 阶段的 PostgreSQL 核心表结构、关键约束和关键索引，为后续 `GW-01`、`PS-01`、`CG-03`、`HI-03`、`KI-03`、`LLM-03` 等任务提供统一数据基线。

## 2. 设计边界

本轮只覆盖：

- `projects`
- `documents`
- `requirements`
- `symbols`
- `symbol_edges`
- `commits`
- `commit_symbols`
- `document_links`
- `model_profiles`
- `query_logs`

本轮不覆盖：

- Redis 键规范
- Qdrant 集合规划
- 统计报表类表结构
- 认证用户实体表

## 3. 通用约定

- 所有业务表统一位于 `app` schema
- 主键统一使用 `bigserial`
- 跨表引用统一使用 `bigint`
- 时间字段统一使用 `timestamptz`
- 表名和字段名统一使用英文小写下划线
- 大多数业务表通过 `project_id` 实现项目级隔离

## 4. 表级数据字典

### 4.1 `app.projects`

用途：项目注册、仓库配置和基础状态管理。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 项目主键 |
| `name` | `varchar(128)` | `not null`, unique | 项目名称 |
| `repo_url` | `varchar(512)` | `not null` | 代码仓库地址 |
| `default_branch` | `varchar(128)` | `not null` | 默认分支 |
| `language_stack` | `varchar(256)` | nullable | 语言栈描述 |
| `doc_repo_path` | `varchar(512)` | nullable | 文档仓或文档目录路径 |
| `status` | `varchar(32)` | `not null` | 项目状态 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |
| `updated_at` | `timestamptz` | `not null` | 更新时间 |

状态范围：

- `CREATED`
- `INDEXING`
- `READY`
- `FAILED`
- `DISABLED`

### 4.2 `app.documents`

用途：保存 Markdown 文档与知识库元数据。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 文档主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `doc_path` | `varchar(512)` | `not null` | 文档相对路径 |
| `doc_type` | `varchar(64)` | `not null` | 文档类型 |
| `title` | `varchar(256)` | `not null` | 文档标题 |
| `metadata_json` | `jsonb` | `not null` | frontmatter 或补充元数据 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |
| `updated_at` | `timestamptz` | `not null` | 更新时间 |

唯一约束：

- `(project_id, doc_path)`

### 4.3 `app.requirements`

用途：保存需求条目及其来源文档关联。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 需求主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `requirement_code` | `varchar(128)` | `not null` | 需求编号 |
| `title` | `varchar(256)` | `not null` | 需求标题 |
| `status` | `varchar(32)` | `not null` | 需求状态 |
| `source_doc_id` | `bigint` | FK, nullable | 来源文档 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |
| `updated_at` | `timestamptz` | `not null` | 更新时间 |

状态范围：

- `DRAFT`
- `ACTIVE`
- `ARCHIVED`
- `DONE`

唯一约束：

- `(project_id, requirement_code)`

### 4.4 `app.symbols`

用途：保存代码图谱符号节点。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 符号主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `file_path` | `varchar(512)` | `not null` | 文件路径 |
| `symbol_name` | `varchar(256)` | `not null` | 符号名称 |
| `symbol_kind` | `varchar(64)` | `not null` | 符号类型 |
| `signature` | `text` | nullable | 符号签名 |
| `line_start` | `integer` | `not null` | 起始行 |
| `line_end` | `integer` | `not null` | 结束行 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |

校验规则：

- `line_start > 0`
- `line_end >= line_start`

### 4.5 `app.symbol_edges`

用途：保存符号关系边。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 关系主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `source_symbol_id` | `bigint` | FK, `not null` | 源符号 |
| `target_symbol_id` | `bigint` | FK, `not null` | 目标符号 |
| `edge_type` | `varchar(32)` | `not null` | 关系类型 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |

关系类型范围：

- `CALLS`
- `IMPORTS`
- `IMPLEMENTS`
- `EXTENDS`
- `REFERENCES`

唯一约束：

- `(project_id, source_symbol_id, target_symbol_id, edge_type)`

### 4.6 `app.commits`

用途：保存 Git 提交记录。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 提交主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `commit_hash` | `varchar(64)` | `not null` | 提交哈希 |
| `author` | `varchar(128)` | `not null` | 作者 |
| `commit_time` | `timestamptz` | `not null` | 提交时间 |
| `message` | `text` | `not null` | 提交信息 |
| `branch_name` | `varchar(128)` | `not null` | 分支名 |
| `created_at` | `timestamptz` | `not null` | 入库时间 |

唯一约束：

- `(project_id, commit_hash)`

### 4.7 `app.commit_symbols`

用途：保存提交与符号的影响映射。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 记录主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `commit_id` | `bigint` | FK, `not null` | 提交主键 |
| `symbol_id` | `bigint` | FK, `not null` | 符号主键 |
| `change_type` | `varchar(32)` | `not null` | 变更类型 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |

变更类型范围：

- `ADDED`
- `MODIFIED`
- `DELETED`
- `RENAMED`
- `UNKNOWN`

唯一约束：

- `(project_id, commit_id, symbol_id, change_type)`

### 4.8 `app.document_links`

用途：保存文档与代码、提交、需求之间的追溯关系。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 记录主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `document_id` | `bigint` | FK, `not null` | 文档主键 |
| `symbol_id` | `bigint` | FK, nullable | 关联符号 |
| `commit_id` | `bigint` | FK, nullable | 关联提交 |
| `requirement_id` | `bigint` | FK, nullable | 关联需求 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |

校验规则：

- `symbol_id`、`commit_id`、`requirement_id` 至少一项非空

### 4.9 `app.model_profiles`

用途：保存项目级模型策略。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 配置主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `provider` | `varchar(64)` | `not null` | 模型提供方 |
| `base_url` | `varchar(512)` | nullable | 网关地址 |
| `model_name` | `varchar(128)` | `not null` | 主模型名 |
| `embedding_model` | `varchar(128)` | nullable | 向量模型名 |
| `timeout_seconds` | `integer` | `not null` | 超时秒数 |
| `fallback_model` | `varchar(128)` | nullable | 兜底模型 |
| `enable_local_only` | `boolean` | `not null` | 是否仅允许本地模型 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |
| `updated_at` | `timestamptz` | `not null` | 更新时间 |

唯一约束：

- `project_id`

### 4.10 `app.query_logs`

用途：保存查询与模型调用基础审计日志。

| 字段 | 类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| `id` | `bigserial` | PK | 日志主键 |
| `project_id` | `bigint` | FK, `not null` | 所属项目 |
| `user_id` | `varchar(128)` | `not null` | 发起人标识 |
| `query_type` | `varchar(64)` | `not null` | 查询类型 |
| `query_text` | `text` | `not null` | 查询原文 |
| `model_used` | `varchar(128)` | nullable | 使用的模型 |
| `cost_token` | `bigint` | `not null` | token 消耗 |
| `created_at` | `timestamptz` | `not null` | 创建时间 |

校验规则：

- `cost_token >= 0`

## 5. 关键外键关系

- `documents.project_id -> projects.id`
- `requirements.project_id -> projects.id`
- `requirements.source_doc_id -> documents.id`
- `symbols.project_id -> projects.id`
- `symbol_edges.project_id -> projects.id`
- `symbol_edges.source_symbol_id -> symbols.id`
- `symbol_edges.target_symbol_id -> symbols.id`
- `commits.project_id -> projects.id`
- `commit_symbols.project_id -> projects.id`
- `commit_symbols.commit_id -> commits.id`
- `commit_symbols.symbol_id -> symbols.id`
- `document_links.project_id -> projects.id`
- `document_links.document_id -> documents.id`
- `document_links.symbol_id -> symbols.id`
- `document_links.commit_id -> commits.id`
- `document_links.requirement_id -> requirements.id`
- `model_profiles.project_id -> projects.id`
- `query_logs.project_id -> projects.id`

## 6. 关键索引

- `idx_symbols_project_symbol_name`
- `idx_symbols_project_file_path`
- `idx_symbol_edges_project_source`
- `idx_symbol_edges_project_target`
- `idx_commits_project_commit_time`
- `idx_commit_symbols_project_symbol`
- `idx_requirements_project_code`
- `idx_documents_project_doc_path`
- `idx_document_links_project_document`
- `idx_query_logs_project_created_at`
- `idx_query_logs_project_query_type`

## 7. 对后续任务的直接支撑

- `GW-01`：可基于 `query_logs` 与 `projects` 继续接审计与项目上下文
- `PS-01`、`PS-02`：直接复用 `projects`
- `CG-03`：直接复用 `symbols` 与 `symbol_edges`
- `HI-03`：直接复用 `commits` 与 `commit_symbols`
- `KI-03`：直接复用 `documents`、`requirements` 与 `document_links`
- `LLM-03`：直接复用 `model_profiles` 与 `query_logs`
