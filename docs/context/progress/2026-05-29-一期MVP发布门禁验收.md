## 本次新增能力（2026-05-29 第二轮推进）

### PS-04 异步索引触发与项目状态机
- `@EnableAsync` + `IndexOrchestratorService` 异步执行
- 项目状态流转：CREATED → INDEXING → READY/FAILED
- `app.projects` 新增 `index_error` 字段，错误时自动写入失败原因
- `POST /api/projects/{id}/sync-indexes` 改为异步（立即返回 INDEXING）

### CG-02~04 CodeGraph Runner 完整索引管线
- 新增 REST API：`POST /api/runner/codegraph/index`
- 符号提取：Java(Kotlin)/TypeScript(JavaScript)/Python 三类语言
- 边构建：IMPORTS（跨文件引用）+ CALLS（同文件方法调用）
- 结果写入 `app.symbols` + `app.symbol_edges`，每次全量重建

### HI-02~04 History Indexer 完整索引管线
- 新增 REST API：`POST /api/runner/history/index`
- Git 历史扫描：git log 解析（hash/author/time/message/branch）
- 需求码提取：正则匹配 `XX-00` 格式编号
- 结果写入 `app.commits`（支持 upsert），需求码关联记录

### KI-02~04 Knowledge Indexer 完整索引管线
- 新增 REST API：`POST /api/runner/knowledge/index`
- 文档扫描：递归遍历 `.md` / `.markdown` 文件
- Frontmatter 解析：YAML 键值对提取并序列化为 JSON
- 需求码自动提取 + `app.requirements` 写入（支持 upsert）
- 文档-需求关联写入 `app.document_links`

## 微服务 REST API 端点汇总

| 服务 | 端点 | 方法 | 说明 |
|------|------|------|------|
| codegraph-runner | `/api/runner/codegraph/index` | POST | 触发代码图谱索引（projectId + workspaceRoot） |
| history-indexer | `/api/runner/history/index` | POST | 触发历史索引（projectId + workspaceRoot + maxCommits） |
| knowledge-indexer | `/api/runner/knowledge/index` | POST | 触发知识索引（projectId + workspaceRoot + docRepoPath） |
| project-service | `/api/projects/{id}/sync-indexes` | POST | 异步编排触发全部三类索引 |
