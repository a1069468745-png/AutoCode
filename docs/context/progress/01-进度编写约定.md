# 进度编写约定

## 目的

本约定用于限制进度文档只记录历史事实、关键实现和验证结论，避免与 `docs/context/current/` 的当前态信息重复。

## 推荐结构

- 本次目标
- 已完成内容
- 关键实现（如适用）
- 验证结论
- 归档说明

## 禁止事项

- 不在进度文档中长期维护 `当前状态`
- 不在进度文档中长期维护 `下一步建议`
- 不在进度文档中长期维护 `风险等级`
- 不在进度文档中长期维护 `下一顺位任务`

以上动态信息应只保留在 `docs/context/current/`。

## 验证记录要求

- 保留验证结论即可
- 非必要不展开调试过程、红绿灯过程和完整命令流水
- 如需保留详细排障过程，优先写入专项设计、计划或代码变更说明，而不是长期堆在进度文档中

## 归档说明要求

- 可以说明“本任务已完成”或“本次推进已结束”
- 可以说明历史衔接关系
- 不应使用会与当前事实竞争的措辞，例如“当前进入”“当前主任务”“下一步建议”

## 命名规则

- 文件名使用 `YYYY-MM-DD-<任务编号或主题>-进度.md`
- 同一任务如需多次记录，优先在主题中保留稳定任务编号，再追加必要限定词
- 不得为了单次聊天补充随意创建新的近义进度文件
