# 2026-05-28 Agent 约束与文档拆分决策

## 决策主题

为 AutoCode 项目建立强约束 Agent 工作协议，并将文档拆分为可按需加载的分层结构。

## 已采纳决策

1. 采用 `AGENT.md + CLAUDE.md` 双入口模式
2. 采用 `indexes / governance / specs / plans / quality / context` 六层文档结构
3. 新会话默认仅读取最小入口集，不全量读取所有文档
4. 严格执行 `先计划后开发`
5. 所有重要工作必须可被进度文档和审计规则追溯
6. 可复用的 Agent 行为优先 skill 化

## 决策原因

- 兼顾强约束与 token 控制
- 支持多会话和多人协作
- 降低对聊天上下文的依赖
- 为后续测试、质量和审计预留稳定边界

## 影响范围

- 文档目录结构
- 新会话启动方式
- 实施和验证流程
- 后续研发协作规范
