# Docker 基础组件初始化验证与阶段切换决策

## 1. 决策背景

在严格执行“先计划后开发”的前提下，当前总执行计划中的 `INF-05` 依赖 `INF-02`、`INF-03`、`INF-04`，不适合直接把本次容器初始化工作认定为正式 `INF-05` 完成。

同时，项目需要先验证本地基础容器和 Windows 一键脚本，确保下一阶段具备稳定研发底座。

## 2. 本次决策

### 决策 1：新增受控预备任务

- 新增 `Docker 基础组件初始化验证` 任务。
- 该任务只覆盖：
  - `deploy/`
  - Docker Compose
  - Nginx 配置
  - PostgreSQL 初始化脚本
  - Windows 启停脚本
  - 健康检查验证

结论：

- 该任务是正式基础设施模块实施前的受控预备任务
- 该任务不等同于正式完成 `INF-05`

### 决策 2：脚本执行层优先保证 Windows 稳定性

- 文档继续统一使用中文
- 执行脚本改用 ASCII 提示，避免 `powershell.exe` 读取 UTF-8 中文时出现解析异常

### 决策 3：Nginx 开发入口默认端口使用 `8080`

原因：

- 降低本机 `80` 端口冲突风险
- 便于后续在内网服务器或反向代理场景下再统一调整

### 决策 4：治理阶段完成并切换阶段

- `治理基线建设阶段` 已完成
- 当前项目进入 `一期 MVP 实施准备阶段`

## 3. 对后续工作的影响

- 后续基础设施模块可直接复用 `deploy/` 当前成果
- 下一步应按模块任务入口启动：
  - `INF-01`
  - `INF-02`
  - `INF-03`
- 在正式进入 `INF-05` 时，应基于当前 `deploy/` 成果扩展，不重做底座
