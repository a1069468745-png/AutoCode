# 会话与上下文加载规则

## 1. 默认最小加载集

每个新会话默认只加载：

1. `AGENT.md`
2. `CLAUDE.md`
3. `docs/indexes/00-总索引.md`
4. `docs/indexes/02-当前阶段索引.md`
5. `docs/indexes/03-当前任务索引.md`

## 2. 按任务类型追加加载

### 规划类

追加：

- `docs/plans/execution/06-开发任务拆分与执行计划.md`
- 相关 PRD、HLD、LLD 片段

### 开发类

追加：

- `docs/plans/tasks/` 下对应模块任务文件
- 相关 LLD 片段
- `docs/quality/testing/01-测试与验证策略.md`

### 测试类

追加：

- `docs/plans/tasks/` 下对应模块任务文件
- `docs/quality/testing/01-测试与验证策略.md`

### 评审类

追加：

- `docs/plans/tasks/` 下对应模块任务文件
- `docs/quality/review/01-评审与质量门禁.md`
- 必要时补充 `docs/quality/audit/01-审计与追溯规则.md`

### 发布类

追加：

- 发布相关计划片段
- `docs/quality/audit/01-审计与追溯规则.md`

### 治理类

追加：

- `docs/governance/`
- `docs/context/decisions/00-决策索引.md`
- `docs/context/progress/00-进度索引.md`

## 3. Token 控制规则

- 不默认读取整份 PRD/HLD/LLD
- 优先读索引和当前状态文档
- 只有当任务确实涉及该模块时才读长文档
- 长文档优先按章节读取，不做全量重复加载
- 连续会话优先读取 `docs/context/current/` 与 `docs/context/progress/00-进度索引.md`
- 只有在需要恢复具体实施脉络时，再按需读取对应历史进度或决策文件

## 4. 新会话恢复策略

新会话先回答三个问题：

1. 当前阶段是什么
2. 当前主任务是什么
3. 当前任务做到哪一步了

只有确认这三点后，才进入实施或分析。

## 5. 实施前强制检查

在真正开始任务前，必须再确认：

1. 当前任务是否已有明确边界
2. 当前任务是否已有进入门禁
3. 当前阶段是否已有自测边界或验收边界
4. 本次任务需要更新哪些文档

如果以上任一项缺失，先补上下文、计划、测试或门禁定义，再实施。

研发阶段默认只要求当前任务所需的开发门禁与研发内自测；评审、审计、发布文档仅在进入对应阶段时追加加载。
