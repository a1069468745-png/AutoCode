# 文档按需加载策略

## 每次会话默认加载（最小集，~2KB）

1. `context/current/01-当前阶段与目标.md` — 当前阶段
2. `context/current/02-当前任务状态.md` — 当前任务

读完以上两篇即可回答：当前在哪个阶段、当前任务是什么、做到哪了。

## 按任务类型追加加载

### 规划 / 拆分任务
→ `plans/execution/06-开发任务拆分与执行计划.md`（只读当前阶段相关章节）

### 编码实现
→ `plans/tasks/<模块>.md`（对应模块任务入口）
→ `specs/architecture/03-详细设计-LLD.md`（只读对应模块章节）
→ 需要需求边界时：`specs/product/01-需求文档-PRD.md`（只读对应功能章节）

### 编写测试
→ `quality/testing/01-测试与验证策略.md`

### 评审 / 安全
→ `quality/review/01-评审与质量门禁.md`
→ `quality/audit/02-安全审计与数据脱敏检查清单.md`

### 发布 / 部署
→ `quality/audit/01-审计与追溯规则.md`

### 治理 / 规则变更
→ `governance/01-开发治理规则.md`
→ `governance/02-门禁与审计规则.md`

## Token 控制（强制）

- ❌ 禁止全量读取 PRD / HLD / LLD（每篇 >10KB）
- ❌ 禁止逐篇读取 `progress/` 下全部历史记录
- ✅ 长文档按章节读取，不全文加载
- ✅ `progress/` 仅在需要追溯历史脉络时才按日期读取
- ✅ 先读 `current/` 判断需要什么，再按需加载

## 新会话恢复流程

先读 `current/` 两篇 → 回答"阶段/任务/进度" → 再决定需要追加哪些文档 → 开始工作

## 文档目录约定

| 目录 | 性质 | 何时读 |
|------|------|--------|
| `context/current/` | 当前事实 | 每次会话 |
| `context/progress/` | 历史记录 | 需要追溯时按日期读 |
| `specs/` | 需求+设计权威 | 编码前按章节读 |
| `plans/` | 执行计划+任务 | 规划/拆分时 |
| `governance/` | 工作规则 | 治理变更时 |
| `quality/` | 测试/审计规范 | 测试/评审/发布时 |
