# INF-04 统一配置与环境变量规范 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为后端服务和前端控制台建立统一的配置分层、Profile 约定和环境变量覆盖规则，并补齐说明文档与示例配置。

**Architecture:** 后端各服务统一采用 `application.yml` 模板，以 `SPRING_PROFILES_ACTIVE` 和环境变量占位实现覆盖；前端采用 `.env` / `.env.example` 约定承接 API 地址与运行模式。敏感配置只通过环境变量注入，规范文档集中说明命名、默认值和覆盖优先级。

**Tech Stack:** `Spring Boot 3.3+`、YAML、`.env`、`Vue 3 + Vite`

---

## 1. 任务边界

- 范围内：
  - 创建后端各服务 `src/main/resources/application.yml`
  - 创建 `frontend/web-console/.env.example`
  - 扩展 `deploy/.env.example`
  - 新增统一配置与环境变量规范说明文档
- 范围外：
  - 不接入企业配置中心
  - 不实现真实数据库访问代码
  - 不扩展正式 Docker Compose 服务定义

## 2. 验证方式

- `Test-Path` 验证各模板文件存在
- `mvn -q -DskipTests package` 验证后端骨架未被配置文件破坏
- `pnpm.cmd build` 验证前端骨架可继续构建
