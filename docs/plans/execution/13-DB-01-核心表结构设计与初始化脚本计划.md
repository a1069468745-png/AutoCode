# DB-01 核心表结构设计与初始化脚本 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 在 PostgreSQL 中落地一期 MVP 所需的核心业务表、关键约束与关键索引，并提供可在空库中顺序执行成功的初始化脚本。

**Architecture:** 延续当前 `deploy/postgres/init/` 目录约定，保留 `001_bootstrap.sql` 负责 schema 与平台标记，再新增 `002_schema.sql` 和 `003_index.sql`。所有表统一放在 `app` schema 下，以 `projects` 为项目隔离根节点，通过外键与关键索引为后续项目管理、代码图谱、历史索引、知识索引和模型配置服务提供稳定底座。

**Tech Stack:** `PostgreSQL 16`、`SQL`、`Docker`

---

## 1. 任务边界

- 范围内：

  - 新增 `deploy/postgres/init/003_index.sql`

### Task 2: 落地关键索引脚本

**Files:**
- Create: `deploy/postgres/init/003_index.sql`

- [ ] **Step 1: 为符号与边关系建立核心查询索引**

```sql
create index if not exists idx_symbols_project_symbol_name on app.symbols (project_id, symbol_name);
create index if not exists idx_symbol_edges_project_source on app.symbol_edges (project_id, source_symbol_id);
```

- [ ] **Step 2: 为历史、需求、文档和审计表建立关键索引**

```sql
create index if not exists idx_commits_project_commit_time on app.commits (project_id, commit_time desc);
create index if not exists idx_query_logs_project_created_at on app.query_logs (project_id, created_at desc);
```
