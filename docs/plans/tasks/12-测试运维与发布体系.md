# 测试、运维与发布体系模块任务入口

## 1. 对应总计划

- `OPS-01` 到 `OPS-04`
- `REL-01` 到 `REL-04`
- `IT-01` 到 `IT-10`

## 2. 模块目标

建立从健康检查、日志、测试、联调到发布门禁的统一质量保障体系。

## 3. 任务边界

- 范围内：可观测性、自动化测试基线、索引监控、审计与脱敏、发布基线、阶段验收、集成联调
- 范围外：不做企业级高可用平台治理

## 4. 进入门禁

- 已明确当前属于测试、联调还是发布任务
- 已明确需要验证的阶段边界或模块边界
- 已明确输出的验收证据形式

## 5. 关键验证

- 服务可探活
- 测试或联调链路可复现
- 发布门禁有证据闭环

## 6. 关键回写

- `docs/context/progress/`
- 阶段验收结果和审计结果
