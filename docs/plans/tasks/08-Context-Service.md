# Context Service 模块任务入口

## 1. 对应总计划

- `CS-01` 到 `CS-08`

## 2. 模块目标

对外提供统一查询编排和最小必要上下文构建能力。

## 3. 任务边界

- 范围内：查询意图识别、三类查询适配器、统一接口、上下文聚合、缓存、调用关系与历史分析查询、文档回溯查询、后续混合检索
- 范围外：不直接替代模型侧智能分析

## 4. 进入门禁

- 图谱、历史、文档三类数据源已有可用数据
- 已明确当前任务属于哪类查询能力
- 已明确是否涉及模型调用

## 5. 关键验证

- 查询接口统一
- 结构化事实包完整
- 调用关系、历史、文档回溯结果可人工校验

## 6. 关键回写

- `docs/context/progress/`
- 查询模型变化时更新执行计划或架构文档
