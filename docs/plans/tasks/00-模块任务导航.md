# 模块任务导航

## 1. 目的

本目录用于给新会话提供“模块级轻量入口”，减少从总执行计划中二次定位的成本。
本目录不是第二套事实源，模块任务文件仅做执行入口，权威内容仍以 [总执行计划](D:\project\AutoCode\docs\plans\execution\06-开发任务拆分与执行计划.md) 为准。

## 2. 使用规则

- 新会话进入开发类、测试类、评审类任务时，优先读取对应模块文件
- 如果模块文件与总执行计划冲突，以总执行计划为准
- 如果模块文件缺失，则先补齐模块入口文件，再开始实施
