# 测试文档索引

## 作用

本目录用于区分“长期有效的测试策略”与“阶段性测试清单、验收结果”等一次性产物。

## 读取规则

- 日常研发阶段默认优先读取 `01-测试与验证策略.md`
- 只有进入特定阶段自测或验收时，才读取对应清单和结果文件
- 历史验收结果默认不进入主导航热路径

## 当前文档分类

- 长期策略：`01-测试与验证策略.md`
- 阶段清单：`archive/02-治理基线阶段验收清单.md`
- 历史结果：`archive/03-治理基线阶段验收结果-2026-05-28.md`

## 历史文件说明

- 验收结果和阶段性测试结果会随项目推进持续累积
- 这类文件应优先归档或通过索引进入，而不是长期暴露在主导航中

## 准入与命名规则

- 长期有效的测试规则、策略、模板保留在主层
- 阶段性清单、验收结果、一次性验证结果进入 `archive/`
- 结果性文件优先使用带日期的命名方式，避免与长期规则文件混名
