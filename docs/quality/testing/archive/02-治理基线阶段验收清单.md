# 治理基线阶段验收清单

## 1. 适用范围

本清单用于 `治理基线建设阶段` 的阶段自测边界与验收边界定义。

## 2. 阶段目标

- 建立强约束 Agent 工作协议
- 建立最小启动集与按需加载规则
- 建立模块任务导航与阶段状态恢复能力
- 建立质量、测试、审计入口

## 3. 阶段自测边界

### 3.1 会话启动边界

必须满足：

- 仅通过最小启动集即可回答“当前阶段是什么”
- 仅通过最小启动集即可回答“当前主任务是什么”
- 仅通过最小启动集即可回答“当前任务进行到哪一步”
- 最小启动集之外的文档加载规则明确

### 3.2 规则边界

必须满足：

- 已明确规定“先计划后开发”
- 已明确规定“无门禁不启动任务”
- 已明确规定“阶段实施前必须有自测边界”
- 已明确规定“完成后回写进度和审计记录”
- 已明确规定“不得依赖聊天上下文作为事实源”

### 3.3 文档结构边界

必须满足：

- `docs/` 已按 `indexes / governance / specs / plans / quality / context` 分层
- 至少存在产品、架构、执行、质量、当前状态五类入口
- 当前阶段和当前任务可通过轻量索引快速定位

### 3.4 模块任务入口边界

必须满足：

- 已存在模块任务导航
- 至少已拆出基础设施、数据库、网关、服务、前端、测试运维类任务入口
- 模块任务入口不替代权威设计文档，只承担执行导航职责

## 5. 通过标准

当且仅当以下条件全部满足时，本阶段可视为通过：

- 所有必检项通过
- 最小启动链路可恢复当前状态
- 治理规则与执行路径无明显冲突
- 模块任务入口可支撑后续阶段实施
- 当前任务状态已回写
- 相关进度与决策文档已回写
