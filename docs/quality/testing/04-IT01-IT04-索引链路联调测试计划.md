# IT-01~04 单模块索引链路联调测试计划

## IT-01 项目接入主链路联调

**目标**: 验证项目创建 → 触发索引 → 状态流转全链路

**前置条件**: PS-04 完成，CG/HI/KI 各服务可启动

**测试步骤**:
1. `POST /api/projects` 创建项目（name: "it01-test"）
2. `POST /api/projects/{id}/sync-indexes` 触发索引（传入有效 workspaceRoot）
3. `GET /api/projects/{id}` 轮询 status，验证 CREATED → INDEXING → READY
4. 验证 `symbolCount > 0` 且 `commitCount > 0` 且 `documentCount > 0`

**预期结果**: 项目状态正确流转，三类索引数据均已入库

---

## IT-02 代码图谱链路联调

**目标**: 验证符号提取 → 边构建 → 查询链路

**前置条件**: CG-04 完成，CS-03 完成

**测试步骤**:
1. 准备含 Java/TS 源码的样例工作区
2. `POST /api/runner/codegraph/index` 触发索引
3. 查询 `app.symbols` 验证符号已入库
4. 查询 `app.symbol_edges` 验证边（IMPORTS/CALLS）已入库
5. `POST /api/query/code` 查询某符号，验证可返回调用关系

**预期结果**: 至少一个样例函数可通过 context-service 查到调用关系

---

## IT-03 历史索引链路联调

**目标**: 验证 commit → symbol → 追溯链路

**前置条件**: HI-04 完成，CS-03 完成

**测试步骤**:
1. 准备含 git 历史的样例工作区（至少 3 个 commit）
2. `POST /api/runner/history/index` 触发索引
3. 查询 `app.commits` 验证提交已入库（含 commit_hash/author/time/message）
4. `POST /api/query/history` 查询某文件的变更历史

**预期结果**: 至少一个符号可追溯到最近提交记录

---

## IT-04 文档关联链路联调

**目标**: 验证文档扫描 → frontmatter → 需求 → 代码回溯链路

**前置条件**: KI-04 完成，CS-03 完成

**测试步骤**:
1. 准备含 `.md` 文档和需求码（如 CS-01）的样例工作区
2. `POST /api/runner/knowledge/index` 触发索引
3. 查询 `app.documents` 验证文档已入库（含 title/metadata_json）
4. 查询 `app.requirements` 验证需求码已提取
5. 查询 `app.document_links` 验证文档-需求关联已建立
6. `POST /api/query/knowledge` 按需求码查询，验证可回溯到文档

**预期结果**: 至少一个需求号可回溯到对应文档和关联代码

---

## 执行说明
- 以上测试依赖 PostgreSQL + Redis 运行环境
- 建议在 `integration-tests/` 下为每个 IT 编写独立脚本
- 当前可通过 `run-mvp-smoke.ps1` 间接覆盖部分链路
