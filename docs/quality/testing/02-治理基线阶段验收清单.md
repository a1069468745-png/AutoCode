# 治理基线阶段验收清单

## 1. 适用范围

本清单用于 `治理基线建设阶段` 的阶段自测边界与验收边界定义。
在进入 `一期 MVP 实施准备阶段` 之前，必须先按本清单完成核验。

## 2. 阶段目标

- 建立强约束 Agent 入口
- 建立按需加载文档结构
- 建立先计划后开发执行协议
- 建立质量、测试、审计入口
- 建立新会话快速接续机制

## 3. 阶段自测边界

### 3.1 文档结构边界

必须满足：

- `docs/` 已按 `indexes / governance / specs / plans / quality / context` 分层
- 原有 PRD、HLD、LLD、分析文档、执行计划均可在新目录下找到
- 根目录存在 `AGENT.md` 与 `CLAUDE.md`

### 3.2 新会话恢复边界

必须满足：

- 仅通过最小启动集即可回答“当前阶段是什么”
- 仅通过最小启动集即可回答“当前主任务是什么”
- 仅通过最小启动集即可回答“当前任务进行到哪一步”
- 最小启动集之外的文档加载规则明确

### 3.3 强约束执行边界

必须满足：

- 明确规定“先计划后开发”
- 明确规定“任务开始前检查边界和进入门禁”
- 明确规定“阶段进入前检查自测边界”
- 明确规定“完成后回写进度和审计记录”

### 3.4 文档语言边界

必须满足：

- 入口约束文件为中文
- 治理规则明确规定后续文档默认中文输出

### 3.5 模块任务入口边界

必须满足：

- 存在模块级任务导航文件
- 存在核心模块的轻量任务入口文件
- 模块文件能指向总执行计划作为权威来源

## 4. 自测用例

### 用例 1：新会话最小恢复

- 输入：仅读取 `AGENT.md`、`CLAUDE.md`、`docs/indexes/00-总索引.md`、`docs/indexes/02-当前阶段索引.md`、`docs/indexes/03-当前任务索引.md`
- 期望：
  - 能正确说出当前阶段
  - 能正确说出当前主任务
  - 能正确说出下一步需要读哪些文档

### 用例 2：开发前门禁检查

- 输入：准备开始一个开发任务
- 期望：
  - 会先检查任务边界
  - 会先检查进入门禁
  - 若缺失则停止实施并补文档

### 用例 3：阶段进入前验证

- 输入：准备从治理阶段进入一期 MVP 实施准备阶段
- 期望：
  - 会先检查是否存在阶段验收清单
  - 若清单缺失则先补齐，不直接进入下一阶段

### 用例 4：按需加载控 token

- 输入：一个明确的模块开发任务
- 期望：
  - 优先读取对应模块任务文件
  - 不会默认全量读取 PRD/HLD/LLD

## 5. 验收结论模板

- 结构验收：通过 / 不通过
- 会话恢复验收：通过 / 不通过
- 约束执行验收：通过 / 不通过
- 模块任务入口验收：通过 / 不通过
- 综合结论：可进入下一阶段 / 不可进入下一阶段

## 6. 阶段出口条件

只有以下条件同时满足时，才允许进入下一阶段：

- 本清单所有必检项通过
- 当前任务状态已回写
- 相关进度与决策文档已回写
