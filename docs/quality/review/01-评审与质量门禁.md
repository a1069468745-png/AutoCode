# 评审与质量门禁

本文件仅在进入评审阶段时生效，不作为日常研发阶段的前置阻塞条件。
评审结论、阶段性评审结果等结果性产物不应长期与本规则文件同层混放，应优先归档或通过索引进入。

命名约束：

- 规则类文档保留稳定序号命名
- 评审结论或阶段性评审结果使用带日期文件名，并优先进入 `archive/`

## 1. 评审关注点

- 是否遵循任务边界
- 是否符合已存在架构和计划
- 是否引入未记录的范围扩张
- 是否补齐必要验证
- 是否完成研发阶段要求的最小回写和必要留痕

## 2. 评审门禁

必须检查：

- 是否有上游计划依据
- 是否有对应实现或文档更新
- 是否有研发阶段验证结论
- 是否有风险说明

## 3. 退回条件

以下情况应退回：

- 先开发后补计划
- 任务边界不清
- 未按权威文档执行
- 无验证结果
- 无回写记录
