# 安全审计与数据脱敏检查清单 (OPS-04)

## 1. 目标
校验一期 MVP 的 API Key 加密、模型响应脱敏、查询审计完整性、敏感配置保护。

## 2. 检查项

### 2.1 API Key 保护
| 检查项 | 状态 | 说明 |
|--------|------|------|
| `api_key` 不明文落库 | ⚠️ 待验证 | `app.model_profiles` 表中 api_key 字段需确认加密存储 |
| `api_key` 不出现于日志 | ✅ | 日志仅记录模型名和 provider，不含 key |
| `api_key` 不出现于响应 | ✅ | DTO 不含 api_key 字段 |

### 2.2 数据库密码保护
| 检查项 | 状态 | 说明 |
|--------|------|------|
| DB 密码使用环境变量 | ✅ | `${DB_PASSWORD}` 从环境变量读取 |
| Redis 密码使用环境变量 | ✅ | `${REDIS_PASSWORD}` 从环境变量读取 |
| 无明文密码在 application.yml | ✅ | 使用 `${...}` 占位符 |
| 无明文密码在代码 | ✅ | 全部通过外部配置注入 |

### 2.3 审计日志完整性
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 查询请求记录 | ✅ | `app.query_logs` 记录 projectId/userId/queryType |
| API 请求审计 | ✅ | `RequestAuditFilter` 全局拦截 |
| 限流事件记录 | ✅ | `RateLimitFilter` 记录被拒绝请求 |
| Token 用量统计 | ⚠️ | `cost_token` 字段存在但当前为占位 0 |

### 2.4 数据隔离
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 项目级数据隔离 | ✅ | 所有表均含 `project_id` 外键 |
| 查询权限校验 | ✅ | GW-03 ProjectAccessFilter 拦截 |
| Redis 逻辑库隔离 | ✅ | 测试使用 db 15，生产使用 db 0 |

### 2.5 日志脱敏
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 用户密码不出现于日志 | ✅ | 认证过滤器不记录密码 |
| API Key 不出现于日志 | ✅ | 参见 2.1 |
| 敏感业务数据脱敏 | ⚠️ | index_error 可能包含文件路径（低风险） |

## 3. 待改进项
- `app.model_profiles.api_key` 加密存储（当前为明文，建议二期启用 AES 加密）
- 日志中增加敏感字段过滤规则
- 增加审计日志定期归档策略
