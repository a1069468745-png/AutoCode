# AutoCode Claude 执行协议

## 1. 角色定位

本文件用于约束 Claude/Codex 类 Agent 在本仓库中的执行方式。
它是执行协议，不是产品需求或架构说明。

## 2. 默认会话启动集

每个新会话默认先读取以下最小集合：

1. `AGENT.md`
2. `CLAUDE.md`
3. `docs/indexes/00-总索引.md`
4. `docs/indexes/02-当前阶段索引.md`
5. `docs/indexes/03-当前任务索引.md`

默认不得全量读取全部设计文档。

## 3. 任务类型分流

完成启动后，先将任务归类为以下一种：

- 查询类
- 规划类
- 开发类
- 测试类
- 评审类
- 发布类
- 治理类

然后只加载该类任务需要的文档。

## 4. 按需加载规则

### 查询类任务

读取：

- 当前任务索引
- 相关执行计划片段
- 必要时读取相关架构片段

### 规划类任务

读取：

- 执行计划
- 当前阶段索引
- 相关 PRD、HLD、LLD 片段

### 开发类任务

读取：

- 当前任务索引
- 目标模块执行计划片段
- 相关 LLD 片段
- 治理门禁规则
- 测试与审计规则

### 测试类任务

读取：

- 当前任务索引
- 目标模块执行计划片段
- `docs/quality/testing/`

### 评审类任务

读取：

- 当前任务索引
- 目标模块计划片段
- `docs/quality/review/`

### 发布类任务

读取：

- 当前阶段索引
- 执行计划中的发布门禁片段
- `docs/quality/audit/`

### 治理类任务

读取：

- `docs/governance/`
- 相关决策与进度文件

## 5. 先计划后开发强制规则

在任何实现工作开始前，必须确认以下全部成立：

- 当前任务有明确边界
- 当前任务已有计划或计划片段
- 所需上游门禁已经完成
- 当前任务已通过进入门禁检查
- 本次任务的验证方式已知
- 本次任务的回写目标已知

如果有任一项缺失，必须先停止实施并补齐计划上下文。

在进入一个新阶段前，还必须确认：

- 当前阶段存在自测边界或验收边界
- 当前阶段的关键验证用例已存在，或已被明确补充

如果阶段自测边界缺失，必须先补测试用例、验证规则或验收边界，再进入阶段实施。

## 6. 实施协议

对每个实现任务，必须执行以下步骤：

1. 先在内部重述任务边界
2. 确认计划、进入门禁和阶段自测边界状态
3. 只在声明边界内实施
4. 运行验证
5. 按边界回写所有应更新文档、进度和审计记录

不得无提示扩大范围。

## 7. 验证协议

任何“已完成”声明，都必须至少有以下一种证据：

- 自动化测试通过
- 构建成功
- 集成验证通过
- 文档一致性检查通过

如果本次没有执行验证，必须明确说明未执行原因。

## 8. 进度同步协议

完成有意义的工作后，应更新：

- `docs/context/progress/`，记录本次变更
- `docs/context/current/02-当前任务状态.md`，记录当前任务状态变化
- `docs/context/decisions/`，如果形成了持久性决策

更新要求：

- 必须更新所有受当前任务影响且按规则应更新的文档
- 不得无依据修改无关文档
- 不得只更新聊天说明而不更新仓库文档

## 9. Token 控制规则

- 不默认全量读取所有规格说明
- 优先读取索引和当前状态文档
- 只读取当前任务真正需要的章节
- 连续会话优先依赖 `docs/context/current/` 和 `docs/context/progress/`
- 会话加载矩阵以 `docs/governance/03-会话与上下文加载规则.md` 为准

## 10. 文档语言规则

- 后续生成的新文档默认使用中文
- 需求、设计、治理、计划、测试、评审、审计、进度、决策类文档必须默认使用中文
- 技术名词、接口名、协议名、字段名、代码标识在必要时可保留英文
- 不得无理由生成英文长篇说明文档

## 11. 质量、测试与审计兼容性

Agent 的执行方式必须与后续以下能力兼容：

- 测试门禁
- 评审门禁
- 审计留痕
- 发布门禁

这意味着不得走未记录的捷径，不得依赖隐藏状态，也不得把关键决策只留在聊天里。
